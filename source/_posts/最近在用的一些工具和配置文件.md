# 前言
最近这个AI领域是越来越火了，连我也忍不住多玩几把。LLM发展真是一浪盖过一浪，一边学底层一边学AI，[VibeCoding](https://x.com/karpathy/status/1886192184808149383)、[MCP](https://modelcontextprotocol.io/)，还有新出的[ACP](https://agentclientprotocol.com/)。[Claude Code](https://docs.anthropic.com/en/docs/claude-code)、[CC-Switch](https://github.com/farion1231/cc-switch)直接绕过，再加上内置插件和AI IDE，真是应接不暇了。在时代里的一粒沙，真的也是个人的一座山啊。

不过这期不是写AI的，我以后也不会写怎么怎么搞一个CC-Switch + Claude Code + DeepSeek API的教学之类的东西，因为我觉得那算不上教学，只是经验积累起来之后就能达到的效果，就像不会说自己在用什么工具。

而是安利一下我最近在用的工具。

本文在 [OpenCode](https://opencode.ai/) 辅助下写作。

# 正文

## 终端

[Alacritty](https://alacritty.org/) 是我目前常用的终端，Rust编写，OpenGL渲染，配置之后很漂亮。我额外Merge了几个Feature过来，比如和 [Neovide](https://neovide.dev/) 同款的动态Cursor，而且跟进了最新的几个Patch。

因为是GPU渲染，所以非常丝滑，而且占用比Windows Terminal低，这就够我选了。

遗憾的是，不支持窗口分割和多开以及标签页功能，倒是像一个纯粹的终端了。

![alt text](../images/最近在用的一些工具和配置文件/image-1.png)


## Linux

实际上有了 [WSL 2](https://learn.microsoft.com/zh-cn/windows/wsl/) 的Windows就是最好的Linux发行版，事实上也是这样的，WSL在不需要图形化开发的场景下简直碾压一切了。

除此之外呢，我给它换了个[XanMod Kernel](https://github.com/Locietta/xanmod-kernel-WSL2)

![alt text](../images/最近在用的一些工具和配置文件/image-2.png)

## 输入法

Windows上的输入法我只觉得 [Rime](https://rime.im/) 好用，我的配置文件在[这个仓库](https://github.com/HowXu/rime)，有基于白霜拼音和基于薄荷拼音的两款。

后台占用只有80m左右，CPU占用更低，几乎是无感而且适配百分之九十九场景使用。

最大的优点是所有快捷键可以自己拿Lua写，可以disable掉Shift了。

![alt text](../images/最近在用的一些工具和配置文件/image-3.png)

## Shell

强力推荐 [Nushell](https://www.nushell.sh/)。依旧Rust编写的神级Shell，依旧所有配置可自定义，依旧有Conda适配(虽然现在Conda用的不多了)。

对于Shell，只要快就可以当世界第一了。

唯一不太好用的就是内置的nu语法，虽然其兼容shell语法，但是这个nu语法有些时候是不太好处理。

![alt text](../images/最近在用的一些工具和配置文件/image-4.png)

配置文件:
``` shell
# alias
alias gcd = git clone --depth=1
alias gcl = git clone

# vfox configuration
source ($nu.default-config-dir | path join "vfox.nu")
$env.config.show_banner = false

```

## 提示符

[Starship](https://starship.rs/)，当然不是马斯克那个：

![alt text](../images/最近在用的一些工具和配置文件/image-5.png)

依旧基于Rust，依旧超快启动速度，搭配起来整个终端只需要不到1s就可以拉起来，而且极其漂亮丝滑美观。

配置文件:
``` toml
format = """

$username\

$conda\
[\ue0b4](fg:#769ff0 bg:#394260)\
$git_branch\
$git_status\
[\ue0b4](fg:#394260 bg:#212736)\
[\ue0b4](fg:#212736 bg:#1d2230)\
$time\
[\ue0b4](fg:#1d2230)\
\n[\u2514](white bold)\
[ \ueb06 at](white bold)\
$directory
$character"""

add_newline = false

style = 'fg:#000000 bg:#91ddff'

[git_branch]
symbol = "\uf418"
style = "bg:#394260"
format = '[[ $symbol $branch ](fg:#769ff0 bg:#394260)]($style)'

[git_status]
style = "bg:#394260"
format = '[[($all_status$ahead_behind )](fg:#769ff0 bg:#394260)]($style)'

format = '[ PyEnv $environment]($style)'

error_symbol = '[❯](bold red)'

```

## 代码编辑器

现在已经是 [LSP](https://microsoft.github.io/language-server-protocol/) 时代了，所以我比较喜欢VSCode的开源版 [VSCodium](https://vscodium.com/)，相当地纯粹，可以一键关闭AI功能。除此之外，我用 [Custom CSS and JS Loader](https://github.com/be5invis/vscode-custom-css) 注入了一些别的特性，比如彩虹标签页，彩虹行号，彩虹提示框描边，Neovide同款Cursor，并且尽量保证整体性能。在正常情况下内存占用在600M左右，是正常的 [Electron](https://www.electronjs.org/) 水平。

## 文本编辑器

[Zed](https://zed.dev/)(又是Rust+GPU)，新时代高性能编辑器，刚上线Windows。总体来说响应很快，而且也可以接LSP，不过整体UI设计实在是难评价，当个 [Notepad++](https://notepad-plus-plus.org/) 平替还是不错的。

## Windows Hook

[Windhawk](https://windhawk.net/)，简单的Windows Hook插件管理器，后台占用极低，但是效果比 [Explorer Patcher](https://github.com/valinet/ExplorerPatcher) 好。

## 魔法

[Clash Verge](https://github.com/Clash-Verge-rev/clash-verge-rev/releases)

这个不多说，依旧Rust依旧GPU加速，而且支持轻量模式(即后台只有Rust和 [Mihomo](https://github.com/MetaCubeX/mihomo) 后端)，几乎无感使用。
